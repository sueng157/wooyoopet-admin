## 공통 레이아웃 규칙
---
| 규칙 | 설명 |
| --- | --- |
| 목록 화면 | 가로로 긴 표 형태. 한 행(row)에 하나의 데이터를 담고, 각 열(column)이 항목이 됨. 페이지당 20건 표시, 하단에 페이지네이션 |
| 상세 화면 | 세로로 긴 표 형태. 영역별로 항목명과 값을 위에서 아래로 나열. 영역 간 구분선 또는 제목으로 구분 |
| 탭 구조 | 하나의 메뉴 안에 탭이 여러 개인 경우, 화면 상단에 탭 버튼을 배치하고 탭 클릭 시 해당 내용으로 전환 |
| 필터·검색 | 목록 화면 상단에 배치. 기간 검색, 상태 필터, 키워드 검색은 한 줄에 가로로 나열 |
| 링크 이동 | "클릭 시 이동"으로 표기된 항목은 해당 텍스트에 하이퍼링크를 걸어 클릭하면 해당 메뉴의 상세 화면으로 이동 |
| 엑셀 다운로드 | 목록 화면 우측 상단에 엑셀 다운로드 버튼 배치. 현재 필터 조건이 적용된 데이터를 다운로드 |
| 편집 버튼 | 목록 화면 맨 오른쪽 열에 배치. 클릭 시 해당 데이터의 상세 화면으로 이동 |
| 가림 처리(마스킹) | 개인정보(휴대폰번호, 호수, 계좌번호 등)는 일부만 표시하고 나머지는 별표(*)로 가림. 목록 화면에서는 마스킹만 표시(전체보기 불가). 상세 화면에서는 마스킹 옆에 [전체보기] 버튼을 배치하여 클릭 시 원본 표시, 다시 클릭 시 [숨기기]로 토글. 전체보기 클릭 시 감사 로그(audit log) 기록 |
| 상태 표시 | 상태값은 색상 라벨(badge)로 구분 (예: 정상=초록, 정지=빨강, 대기=노랑 등). 구체적 색상은 개발 시 결정 |
| 반응형 | 관리자 페이지는 PC 브라우저 기준으로 제작. 최소 너비 1280px 기준 |

## 0. 대시보드
대시보드는 관리자가 로그인하면 가장 먼저 표시되는 화면입니다. 목록·상세 구조가 아닌 단일 화면이며, 4개 영역으로 구성됩니다.
---
### 영역 1: 오늘의 현황
화면 최상단에 가로 한 줄로 카드 6개를 나란히 배치합니다. 각 카드에는 숫자가 크게 표시되고, 카드 클릭 시 해당 메뉴로 이동합니다.
| 카드 | 표시 내용 | 클릭 시 이동 |
| --- | --- | --- |
| 오늘 신규 가입 | 오늘 가입한 회원 수 | 회원관리 목록 (가입일 = 오늘 필터 적용) |
| 오늘 신규 예약 | 오늘 접수된 돌봄 예약 건수 | 돌봄예약관리 목록 (신청일 = 오늘 필터 적용) |
| 오늘 등원 예정 | 오늘 등원이 예정된 돌봄 건수 | 돌봄예약관리 목록 (등원일 = 오늘 필터 적용) |
| 현재 돌봄 진행중 | 지금 이 시점에 진행중인 돌봄 건수 | 돌봄예약관리 목록 (상태 = "돌봄진행중" 필터 적용) |
| 오늘 결제 금액 | 오늘 발생한 결제 총액 | 결제관리 > 결제내역 탭 (결제일 = 오늘 필터 적용) |
| 오늘 취소·환불 | 오늘 발생한 취소/환불 건수 | 결제관리 > 환불/위약금 탭 (요청일 = 오늘 필터 적용) |
---
### 영역 2: 관리자 승인 대기
영역 1 아래, 화면 중앙 왼쪽에 배치합니다. 관리자가 직접 확인하고 처리해야 하는 항목만 모아서 보여줍니다. 각 항목은 건수가 0이면 회색, 1건 이상이면 빨간색 숫자로 강조 표시합니다.
| 항목명 | 표시 내용 | 클릭 시 이동 |
| --- | --- | --- |
| 주소 인증 대기 | 승인·거절되지 않은 주소 인증 요청 건수 | 회원관리 목록 (주소 인증 상태 = "대기" 필터 적용) |
| 정산정보 신규 제출 | 유치원이 제출했지만 이니시스 등록 요청이 아직 진행되지 않은 건수 | 정산관리 > 정산정보 탭 (상태 = "미등록" 필터 적용) |
| 정산정보 등록 실패 | 이니시스 서브몰 등록이 실패하여 재처리가 필요한 건수 | 정산관리 > 정산정보 탭 (상태 = "실패" 필터 적용) |
| 환불 대기 | 환불 요청이 접수되었지만 아직 처리되지 않은 건수 | 결제관리 > 환불/위약금 탭 (상태 = "환불대기" 필터 적용) |
| 신고 미처리 | 접수 또는 처리중 상태인 신고 건수 | 채팅관리 > 신고접수 탭 (상태 = "접수"+"처리중" 필터 적용) |
| 정산 보류 | 정산이 보류 상태인 건수 | 정산관리 > 정산내역 탭 (상태 = "정산보류" 필터 적용) |
| 피드백 미확인 | 관리자가 아직 확인하지 않은 의견/피드백 건수 | 설정 > 의견/피드백 탭 (확인 여부 = "미확인" 필터 적용) |
---
### 영역 3: 이달 매출 요약 (수정본)
영역 1 아래, 화면 중앙 오른쪽에 배치합니다 (영역 2와 나란히). 이번 달 매출 핵심 숫자만 표시합니다.
※ 환불 방식: 위약금을 별도 결제 → 기존 결제 전액 취소. 따라서 취소·환불 금액은 플랫폼 수입에 영향 없음.
| 항목명 | 표시 내용 |
| --- | --- |
| 이번 달 돌봄 결제금액 | 이번 달 정상적으로 완료된 돌봄 결제 총액 (결제취소된 건은 제외) |
| 이번 달 위약금 결제금액 | 이번 달 위약금으로 새로 결제된 총액 |
| 이번 달 총 유효 거래금액 | 돌봄결제금액 + 위약금 결제금액 (실제로 취소되지 않고 남아있는 거래 총액) |
| 이번 달 플랫폼 수수료 수입 | 총 유효 거래금액 × 20% |
| 이번 달 유치원 정산 총액 | 총 유효 거래금액 × 80% |
| 이번 달 취소·환불 금액 | 이번 달 결제취소 처리된 기존 결제건의 총액 (플랫폼 수입 무관, 중개수수료X, 결제수수료X) |
| 전월 대비 증감 | 지난 달 플랫폼 수수료 수입 대비 증가·감소율 (%, 증가면 초록 화살표, 감소면 빨간 화살표) |
---
### 영역 4: 최근 활동 로그
화면 하단에 배치합니다. 서비스에서 가장 최근에 발생한 주요 이벤트 5건을 시간순으로 표시합니다.
| 열 | 설명 |
| --- | --- |
| 발생일시 | 이벤트가 발생한 날짜와 시간 |
| 이벤트 유형 | 신규가입, 유치원등록, 예약접수, 결제완료, 취소요청, 환불완료, 노쇼발생, 신고접수, 정산완료 등 (라벨 색상으로 구분) |
| 내용 요약 | 간략한 설명 (예: "홍길동님이 행복한유치원에 돌봄 예약을 신청했습니다") |
| 관련 링크 | "바로가기" 버튼, 클릭 시 해당 상세 화면으로 이동 |

|  | 종료일 | 다음 버전이 시행되면서 종료된 날짜 |
|  | 수정 사유 | 해당 버전에서 변경된 주요 내용 |
|  | 본문 보기 | 과거 버전의 약관 전문 보기 링크 |
상세 기능: 저장(등록 또는 수정), 새 버전 발행(기존 버전은 자동으로 이력에 보관되고 새 버전이 현재 적용 버전으로 전환), 비공개 전환, 삭제(동의 이력이 있는 약관은 삭제 불가, 비공개로만 전환 가능 경고 표시).
